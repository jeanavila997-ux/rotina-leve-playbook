Resumo selecionado — Awesome Design Resources List

Este resumo reúne recursos selecionados do repositório "Awesome-Design-Resources-List" para uso direto no playbook "Rotina Leve". Arquivos originais (README.md, LICENSE e imagem) foram copiados para content/REFERENCIAS/Awesome-Design-Resources-List.

Observação sobre uso: verifique licenças antes de republicar ativos; o README original foi preservado na pasta de referências.

Principais categorias e recursos recomendados (curadoria rápida para o playbook)

1) Inspiração & Comunidade
- Mobbin — biblioteca de padrões de mobile e web (estudos de caso e telas reais)
- Land-book / Land-book (land-book.com) — galerias de landing pages selecionadas
- Dribbble — inspiração e portfolios

2) UI Kits & Componentes
- Tailwind components — blocos UI prontos para uso
- shadcn/ui — componentes prontos para copiar/colar
- Radix UI — primitives acessíveis para React

3) Cores e Paletas
- Coolors — gerador e exploração de paletas
- Adobe Color — extração e harmonização de cores
- Happy Hues — paletas aplicadas em exemplos reais

4) Tipografia
- Google Fonts — repositório gratuito de fontes
- Fontpair / Fontjoy — pareamento e experimentação de tipos

5) Ícones & Ilustrações
- Heroicons — SVGs práticos para UI
- Font Awesome — kit amplo de ícones
- unDraw / Open Doodles — ilustrações gratuitas e customizáveis

6) Imagens & Backgrounds
- Unsplash / Pexels / Pixabay — bancos de imagens royalty-free
- Remove.bg — remoção automática de fundo (útil para mockups)

7) Ferramentas de Produção & AI
- TinyPNG — compressão de imagens
- Remove.bg / Vectorizer AI — tratamento de imagens para assets
- ChatGPT / Midjourney / Stable Diffusion — ideação e geração visual (use com cuidado de direitos)

8) Plugins e fluxo Figma (destaques para integração com processos)
- Design Lint — encontrar inconsistências de estilos
- Content Reel — gerenciar strings, imagens e ícones no design
- Stark — verificação de contraste e acessibilidade

Como usar este resumo no playbook
- Importar como referência: incluir link para content/REFERENCIAS/Awesome-Design-Resources-List/README.md
- Selecionar 1–2 recursos por capítulo onde for aplicável (ex.: capítulo sobre cores cita Coolors e Happy Hues)
- Validar licenças de cada asset antes da republicação

Se desejar, gero versões específicas por capítulo (ex.: recursos para "cores", para "tipografia", para "prototipação") ou extraio automaticamente os N primeiros links por categoria do README original.
